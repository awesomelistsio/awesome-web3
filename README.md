## Foundations & Concepts

- [Web3 Explained](https://ethereum.org/en/web3/) – Overview of the decentralized web and its core principles.
- [Blockchain Basics](https://www.investopedia.com/terms/b/blockchain.asp) – Foundational concepts behind distributed ledgers.
- [Smart Contracts](https://ethereum.org/en/developers/docs/smart-contracts/) – Programs that run on blockchains and automate agreements.
- [Decentralization](https://www.coindesk.com/learn/what-is-decentralization/) – Core design principle of Web3 systems.
- [Tokenomics](https://www.investopedia.com/terms/t/tokenomics.asp) – Economic design of tokens and incentives.

## Blockchains & Execution Layers

- [Ethereum](https://ethereum.org/) – Programmable blockchain powering most Web3 applications.
- [Solana](https://solana.com/) – High-performance blockchain optimized for throughput.
- [Polkadot](https://polkadot.network/) – Multi-chain platform enabling interoperable blockchains.
- [Avalanche](https://www.avax.network/) – High-speed blockchain supporting custom subnets.
- [NEAR Protocol](https://near.org/) – Developer-friendly blockchain with sharding.
- [Cosmos](https://cosmos.network/) – Ecosystem of interoperable blockchains using IBC.

## Layer 2 & Scaling

- [Arbitrum](https://arbitrum.io/) – Ethereum optimistic rollup for scalable dApps.
- [Optimism](https://www.optimism.io/) – Layer 2 focused on fast and low-cost transactions.
- [zkSync](https://zksync.io/) – Zero-knowledge rollup for Ethereum scalability.
- [StarkNet](https://starknet.io/) – ZK-based Layer 2 for scalable smart contracts.
- [Polygon](https://polygon.technology/) – Scaling and interoperability solutions for Ethereum.

## Smart Contract Development

- [Solidity](https://docs.soliditylang.org/) – Primary programming language for Ethereum smart contracts.
- [Vyper](https://vyper.readthedocs.io/) – Python-inspired language for secure smart contracts.
- [Hardhat](https://hardhat.org/) – Development environment for Ethereum smart contracts.
- [Foundry](https://book.getfoundry.sh/) – Fast Ethereum development toolkit written in Rust.
- [Truffle](https://trufflesuite.com/) – Smart contract development and deployment framework.

## Frontend & dApp Frameworks

- [Web3.js](https://web3js.readthedocs.io/) – JavaScript library for interacting with blockchains.
- [Ethers.js](https://docs.ethers.org/) – Lightweight JavaScript library for Ethereum interactions.
- [RainbowKit](https://www.rainbowkit.com/) – Wallet connection UI for Web3 applications.
- [wagmi](https://wagmi.sh/) – React hooks for Ethereum development.
- [Next.js](https://nextjs.org/) – Popular React framework commonly used for dApps.

## Wallets & Identity

- [MetaMask](https://metamask.io/) – Browser-based wallet and Web3 gateway.
- [WalletConnect](https://walletconnect.com/) – Protocol for connecting wallets to dApps.
- [Ledger](https://www.ledger.com/) – Hardware wallet for secure key storage.
- [ENS](https://ens.domains/) – Decentralized naming system on Ethereum.
- [Ceramic](https://ceramic.network/) – Decentralized identity and data network.

## Storage & Decentralized Data

- [IPFS](https://ipfs.tech/) – Peer-to-peer distributed file system.
- [Filecoin](https://filecoin.io/) – Incentivized decentralized storage network.
- [Arweave](https://www.arweave.org/) – Permanent decentralized data storage.
- [Storj](https://www.storj.io/) – Decentralized cloud object storage.
- [OrbitDB](https://orbitdb.org/) – Distributed peer-to-peer database.

## Interoperability & Messaging

- [LayerZero](https://layerzero.network/) – Omnichain interoperability protocol.
- [Wormhole](https://wormhole.com/) – Cross-chain messaging and bridging protocol.
- [Axelar](https://axelar.network/) – Secure cross-chain communication network.
- [IBC](https://ibcprotocol.org/) – Inter-Blockchain Communication protocol for Cosmos.
- [Connext](https://www.connext.network/) – Cross-chain liquidity and messaging protocol.

## Oracles & External Data

- [Chainlink](https://chain.link/) – Decentralized oracle network for off-chain data.
- [API3](https://api3.org/) – First-party oracle solution for Web3.
- [Pyth Network](https://pyth.network/) – High-frequency financial data oracle.
- [Band Protocol](https://bandprotocol.com/) – Cross-chain data oracle platform.

## Indexing, Analytics & Search

- [The Graph](https://thegraph.com/) – Decentralized indexing protocol for blockchain data.
- [Dune](https://dune.com/) – SQL-based blockchain analytics platform.
- [Etherscan](https://etherscan.io/) – Blockchain explorer for Ethereum.
- [Nansen](https://www.nansen.ai/) – Wallet and on-chain analytics platform.
- [Covalent](https://www.covalenthq.com/) – Unified API for blockchain data.

## Governance, DAOs & Coordination

- [Aragon](https://aragon.org/) – DAO creation and governance framework.
- [DAOstack](https://daostack.io/) – Tools for decentralized governance.
- [Snapshot](https://snapshot.org/) – Off-chain governance and voting platform.
- [Tally](https://www.tally.xyz/) – Governance dashboards for DAOs.
- [OpenZeppelin Governor](https://docs.openzeppelin.com/contracts/governance) – Governance smart contract modules.

## Security, Privacy & Compliance

- [OpenZeppelin](https://www.openzeppelin.com/) – Security libraries and audited smart contracts.
- [Slither](https://github.com/crytic/slither) – Static analysis tool for Solidity.
- [MythX](https://mythx.io/) – Smart contract security analysis platform.
- [CertiK](https://www.certik.com/) – Blockchain security audits and monitoring.
- [zk-SNARKs](https://z.cash/technology/zksnarks/) – Zero-knowledge proof technology for privacy.

## Infrastructure & Node Providers

- [Infura](https://www.infura.io/) – Managed blockchain node infrastructure.
- [Alchemy](https://www.alchemy.com/) – Developer platform for Web3 applications.
- [QuickNode](https://www.quicknode.com/) – High-performance blockchain node provider.
- [Ankr](https://www.ankr.com/) – Decentralized Web3 infrastructure platform.
- [Pocket Network](https://www.pokt.network/) – Decentralized RPC network.
